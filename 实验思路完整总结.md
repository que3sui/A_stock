## 1. 问题定义与目标

### 1.1 任务

预测 A 股每只股票**未来 5 个交易日**的横截面 rank-percentile（相对排序），并据此执行每日调仓的模拟交易。

**核心约束**:

- 必须包含深度学习模型（作业硬性要求）
- 严禁数据泄露（未来函数、全量标准化）
- 2016-2022 训练 / 2023 验证 / 2024-2026.05 测试（严格按时间切分）
- 模拟交易比赛: 2026-06-01 至 2026-06-12（同花顺 FDL2026）

### 1.2 评估指标

| 层级   | 指标                           | 含义                             |
| ------ | ------------------------------ | -------------------------------- |
| 统计层 | IC / RankIC / ICIR             | 预测得分与真实标签的横截面相关性 |
| 收益层 | 年化收益率 / 夏普比率 / Calmar | 风险调整后的策略表现             |
| 风控层 | 最大回撤 / VaR / CVaR          | 极端情况下的损失控制             |
| 归因层 | CAPM Alpha / Information Ratio | 独立于市场的纯 alpha 能力        |

### 1.3 初始假设

1. Transformer 架构因其注意力机制天然适合股票横截面排序
2. 市场状态信息应当调制个股因子（market guidance hypothesis）
3. 多模型集成优于单模型
4. 更多参数 = 更好性能
5. 更多特征 = 更好预测

**重要**: 实验过程中，假设 3/4/5 被系统性地证伪。

## 2. 核心方法论: Doubt → Refactor → Simplify

整个实验过程遵循 **三轮迭代开发循环**:

```
Doubt (质疑) → Refactor (重构) → Simplify (精简)
     ↑                                    ↓
     └──────────── 下一轮 ─────────────────┘
```

### 第一轮: 搭建基线

- **Doubt**: 作业给的框架是否足够？数据质量如何？
- **Refactor**: 完整数据管线（6 步）+ 4 模型并行对比（LGBM / MLP / GRU / MASTER）
- **Simplify**: 确定 20 因子 + 5 日标签 + 中证 800 池为基线配置

### 第二轮: 消融验证

- **Doubt**: 权重因子的信号结构是什么？news_count 真的有用吗？
- **Refactor**: 7 项消融实验，独立权重通道架构
- **Simplify**: 只有双通道设计正向突破，其余 6 项全部回退

### 第三轮: 工程巩固

- **Doubt**: 10 个文件各自定义 FACTOR_COLS，漏一个 = 静默 bug？
- **Refactor**: 提取 config.py / losses.py / metrics.py 共享模块
- **Simplify**: 所有模块单一数据源，消除 4 处重复 IC 聚合 + 4 处重复 loss 定义

### 停止条件

复杂性增加导致性能退化时，回退到更简单架构并记录原因。v4-v7 全部低于 v3_search（Sharpe 2.47），这是系统性教训：**先验证简单方案，再叠加复杂度**。

## 3. 数据管线设计

### 3.1 管线流程

```
daily/ (量价) + metric/ (基本面) + moneyflow/ (资金流) + stock_st/ (ST标记)
  │
  ├─ build_panel.py ──→ panel.parquet (1074万行, 5756股×2524天, 50列)
  ├─ universe.py     ──→ universe.parquet (中证800近似, 月度滚动)
  │
  ├─ factors.py      ──→ factors_raw.parquet (20个原始因子)
  ├─ neutralize.py   ──→ features.parquet (MAD去极值→行业+市值OLS残差→Z-score clip)
  ├─ labels.py       ──→ labels.parquet (5日forward log return + 横截面rank)
  └─ market_features.py → market_features.parquet (13维: 12维市场+news_count, train段标准化)
```

### 3.2 关键设计决策

**为什么选中证 800 近似池而非全 A 股 (~5000 只)？**

1. **流动性**: 全 A 股含约 1500 只微盘股（日均成交 < 5000 万），无法保证模拟交易成交
2. **与比赛匹配**: FDL2026 限 "除 ST/北交所"，实际机构资金集中在中大盘股
3. **计算效率**: 5000 股 × 2500 天 = 1250 万样本，MASTER 在 RTX 5070 上受限于 ~1.7 GB GPU 内存
4. **月度滚动避免生存偏差**: 每月用当月可见 circ_mv 重新选，月换手约 4.6%

**为什么是 5 日 forward label？**

- 1 日标签噪声过大（日收益率信噪比低）
- 20 日标签样本量不足（非重叠窗口）
- 5 日标签在信噪比和样本量间取平衡，且与作业要求的 10 天比赛周期匹配（model → trade 5 days, 比赛正好 2 个周期）

### 3.3 数据切分

| 子集 | 时间              | 交易日 | 用途                  |
| ---- | ----------------- | ------ | --------------------- |
| 训练 | 2016-01 ~ 2022-12 | ~1700  | 模型参数学习          |
| 验证 | 2023-01 ~ 2023-12 | ~240   | 超参选择 + early stop |
| 测试 | 2024-01 ~ 2026-05 | ~600   | 最终评估（严格 OOS）  |

**为什么不随机打乱？** 金融数据的时间依赖性意味着随机打乱 = 数据泄露。时间序列的前后依赖性要求严格按时间切分。

## 4. 特征工程: 双通道架构的发现之路

### 4.1 因子体系 (20 维)

| 类别       | 因子                                      | 理论基础                                   |
| ---------- | ----------------------------------------- | ------------------------------------------ |
| 动量 (4)   | mom_5, mom_20, mom_60, mom_120            | Jegadeesh & Titman (1993), A股"短反中长动" |
| 反转 (2)   | rev_1, rev_5                              | 短期反转效应 (A股特有, 散户追涨杀跌后退潮) |
| 波动率 (2) | vol_20, vol_60                            | 低波动异象 (Ang et al., 2006)              |
| 流动性 (2) | turnover_20, amihud_20                    | Amihud (2002) 非流动性, A股小盘流动性溢价  |
| 资金流 (3) | mf_net_5, mf_lg_strength, mf_elg_strength | 主力资金流向的信息含量                     |
| 基本面 (3) | pe_ttm_rank, pb_rank, circ_mv_log         | Fama-French 价值/规模因子 A股本土化        |
| 技术 (4)   | rsi_14, bias_20, vwap_dev, vol_zscore     | 技术面辅助信号                             |

### 4.2 中性化管线

```
原始因子 → MAD 去极值 (±5σ clip) → 行业dummy + log市值 OLS 残差 → Z-score clip(-5, 5)
```

**为什么中性化？** 量价因子天然携带行业和市值偏差（银行股波动小、大盘股 turnover 低）。中性化提取"纯 alpha"——同样的波动率信号，银行股 vs. 科技股的横截面差异才有选股意义。

### 4.3 双通道架构: 本项目的核心突破

**问题发现**: 将 `hs300_weight` / `hs300_dweight` / `cyb_weight`（指数成分股权重）加入 FACTOR_COLS 并过中性化管线后，MASTER 夏普从 2.00 崩溃到 0.80。

**根因分析**: 权重因子的信号天然集中在大市值金融股——"因为是银行/因为市值大所以权重大"正是中性化要移除的偏差。行业+市值 OLS 残差精确移除了权重信号的来源。

**解决方案 — 独立权重通道**:

```
中性化通道 (20因子): MAD → OLS残差 → Z-score → stock_proj → concat ─┐
权重通道 (3因子):   原始值(不过任何管线) → weight_proj ──────────────┘
                                                                    ↓
                                                          MASTER 主网络
```

**关键设计**:

- Market-gate 仅调制中性化因子，权重因子保留原始横截面信号
- 权重因子必须满足三条件: 月频→日频前向填充 + 不中性化 + 独立通道
- 仅增加 ~400 参数（总参数量 123K 不变）

**效果**: ICIR 年化 6.14→7.29 (+19%)，夏普 2.00→2.35 (+18%)，总收益 74%→127% (+70%)

### 4.4 市场特征 (13 维)

3 指数 × 4 特征 + news_count = 13 维，用训练段 (≤2022) mean/std 标准化（全期标准化 = 数据泄露，已修复）。

## 5. 模型架构演进: 从 v1 到 v7 的完整实验矩阵

### 5.1 模型对比总览

| 模型          | 类型        | 参数量    | 横截面感知      | 核心机制                            |
| ------------- | ----------- | --------- | --------------- | ----------------------------------- |
| LightGBM      | 树模型      | 83 trees  | ✅ 树天然比较   | 梯度提升，特征切分                  |
| MLP           | DL 基线     | ~144K     | ❌ 无           | T×F flatten → 3层MLP                |
| GRU+Att       | DL 时序     | ~43K      | ❌ 仅时序       | 双层GRU + Attention pooling         |
| **MASTER v1** | **DL 双轴** | **~123K** | **✅ 显式建模** | Market-gate + 时序TX + 横截面TX     |
| MASTER v2     | DL 双轴     | ~422K     | ✅              | v1 加深加宽 (反例)                  |
| MASTER v3     | DL 集成     | 123K×3    | ✅              | Multi-seed rank-percentile平均      |
| **v3_search** | **DL 搜索** | **123K**  | **✅**          | **10-seed训练搜索+Top3集成 (SOTA)** |
| MASTER v4     | DL 实验     | ~140K     | ✅              | 时间衰减+行业轮动+动态仓位          |
| MASTER v5     | DL 实验     | ~140K     | ✅              | v4 + 滤波器因子                     |
| MASTER v6     | DL 实验     | ~140K     | ✅              | v5 + 长窗口                         |
| MASTER v7     | DL 实验     | ~123K     | ✅              | Sharpe-aware loss                   |

### 5.2 MASTER 架构详解 (AAAI 2024 简化版)

```
输入: X[N,T,20] 因子 + X_w[N,T,3] 权重通道 + market[T,13] 市场状态

┌─────────────────────────────────────────────┐
│ Market-Gate: sigmoid(market_proj) ⊙ X       │  ← 市场状态调制因子
├─────────────────────────────────────────────┤
│ stock_proj (20→48) + weight_proj (3→16)     │  ← 双通道嵌入
│ concat → [N, T, 64]                         │
├─────────────────────────────────────────────┤
│ Intra-stock Transformer (时序, 2层, 4头)     │  ← 单股内部时序模式
│ Temporal Attention Pooling                   │  ← 压缩时序维度
├─────────────────────────────────────────────┤
│ Inter-stock Transformer (横截面, 1层, 4头)   │  ← 股票间相对排序
│ Residual: Z_t + Z_c                         │
├─────────────────────────────────────────────┤
│ Head: LN → Linear(32) → GELU → Linear(1)    │  ← 输出 score[N]
└─────────────────────────────────────────────┘

By-day batch: 每个batch = 一天所有股票，直接对横截面排序
T=20 (回溯20天), H=64, ~123K参数
```

### 5.3 关键架构决策

**为什么 Intra + Inter 双轴？** GRU 只看单股时序（无横截面）表现最差。LightGBM 虽然无显式横截面建模，但树的特征切分天然做横截面比较——这是为什么一个 83 棵树的简单模型能超越 GRU。MASTER 显式建模 inter-stock 关系，取得 SOTA。

**为什么 Market-Gate 有效？** 将个股 alpha 与大盘 beta 解耦后再融合，比直接堆叠特征更有效。市场状态通过 gate 机制选择性地放大/抑制不同因子。

**为什么 v2 加深加宽反而过拟合？** 参数量 122K→422K (3.5x)，val_rank_ic 微升 0.069→0.070，但 test_rank_ic 显著降 0.057→0.048，夏普 2.24→1.12。**A 股短期 alpha 信号容量上限约 ~100K 参数**——数据信号的复杂度有限，更大模型只是拟合噪声。

**为什么 v3 multi-seed ICIR 升但夏普降？** 3 个 seed 的 rank-percentile 平均减少了方差（ICIR 6.08→6.98 +15%），但同时也稀释了顶部排序的确定性——top-K 选股质量下降。**这是 IC ≠ 回测收益的最有力证据之一**。

### 5.4 v4-v7 实验: 复杂度陷阱

| 版本 | 新增机制                           | 结果               | 教训                                             |
| ---- | ---------------------------------- | ------------------ | ------------------------------------------------ |
| v4   | 时间衰减loss + 行业轮动 + 动态仓位 | Sharpe < v3_search | 行业轮动因子与权重通道信息重叠                   |
| v5   | v4 + 滤波器因子                    | Sharpe < v3_search | 滤波器引入更多参数但信号增益为负                 |
| v6   | v5 + 长窗口 T=30                   | Sharpe < v3_search | 更长窗口稀释了短期反转信号                       |
| v7   | Sharpe-aware loss 替代 IC loss     | Sharpe 1.90        | 直接优化 Sharpe 在训练中不稳定，需 IC stabilizer |

**系统性教训**: v4-v7 全部低于 v3_search（Sharpe 2.47）。每次增量改进方向逻辑上合理但实验效果为负。A 股量价信号的信息量有限——**简单方案 + 正确的特征结构 > 复杂模型**。

### 5.5 v3_search: 当前 SOTA

- 10 个 seed 独立训练 MASTER v1 + 权重通道
- 按 val_rank_ic 筛 top-3
- top-3 分别回测，选夏普最高者
- 最佳种子 314/1024/1337 集成，Sharpe 2.47 (n=10, k=2, 真实约束)

## 6. 损失函数设计

### 6.1 复合损失

```
L = α × IC_loss + (1-α) × TopK_margin_loss,  α=0.6

IC_loss = -Pearson(scores, labels)           # 全样本排序质量
TopK_margin_loss = max(0, s_bot - s_top + m) # top-K 确信度, m=0.1
```

### 6.2 设计理由

- **IC loss**: 优化全横截面的排序质量（Pearson 相关系数），保证整体方向正确
- **Top-K margin loss**: 确保"最好的 K 只"和"最差的 K 只"之间有足够的安全边际
- **α=0.6**: IC loss 为主（保证全样本排序），margin loss 为辅（sharpening top-K）
- **为什么不用 MSE？** MSE 优化逐股预测精度，但我们要的是排序而非精确值
- **为什么不用 Sharpe loss？** v7 实验证明直接优化 Sharpe 训练不稳定，需 IC stabilizer（λ=0.3），最终不如 IC loss 稳定

## 7. 回测引擎: 真实约束的重要性

### 7.1 真实约束清单

| 约束       | 实现方式                                          |
| ---------- | ------------------------------------------------- |
| T+1 隐式   | portfolio 来自上日 score，用当日 pct_chg 计收益   |
| 涨跌停过滤 | 一字板 (open==high==low) 自动跳过                 |
| 双边手续费 | 买入 0.025% 佣金 + 卖出 0.025% 佣金 + 0.1% 印花税 |
| ST 剔除    | 当日 stock_st 数据动态剔除                        |
| 满仓约束   | n=10 等权，每只 1/n=10%，总仓位 100%              |

### 7.2 真实 vs 理想化对比

| 指标     | 理想化 (无费/无涨跌停) | 真实 (有费/有涨跌停) | 变化      |
| -------- | ---------------------- | -------------------- | --------- |
| 年化收益 | 45.2%                  | 42.8%                | -2.4pp    |
| 夏普     | 2.24                   | 2.12                 | -5%       |
| 累积净值 | 116%                   | 79%                  | **-37pp** |

**关键洞察**: 年化和夏普对交易摩擦稳健（-5%），但累积净值因复利累积放大差异（-37pp）。**因此年化和夏普才是策略可行性的稳健度量**，不应只看累积收益。

### 7.3 (n, k) 敏感度分析

4×4 网格 (n=5/10/20/30, k=1/2/3/5) 的回测显示:

- 默认配置 (n=10, k=2, Sharpe 2.12) **不是最优**
- (n=20, k=5) 取得最佳 Sharpe **2.78** 和 Calmar **5.26**
- k=1（低换手）在所有 n 下都最差 — 模型每日刷新信号但无法执行
- n=5 持仓过于集中，单股权重 20% 风险大

## 8. 消融实验: 7 次假设验证

| #     | 实验                              | 夏普变化             | 结论                         |
| ----- | --------------------------------- | -------------------- | ---------------------------- |
| 1     | 权重因子加入 FACTOR_COLS (中性化) | 2.00→0.80 (-60%)     | ❌ 中性化杀信号              |
| **2** | **权重因子独立通道**              | **2.00→2.35 (+18%)** | **✅ SOTA 突破**             |
| 3     | + news_count (市场活跃度)         | 2.35→1.46            | ❌ ICIR 升但夏普降           |
| 4     | + 关键词情感 (mean/std)           | 2.35→1.16            | ❌ 词典法精度不够            |
| 5     | v3 加权集成 (val RIC 权重)        | 2.35→1.32            | ❌ 弱 seed 稀释强 seed       |
| 6     | PE 30 日滞后 (财报修正)           | 2.35→1.63            | ❌ 合规性收益 < 信号时效损失 |
| 7     | + 北向资金 5 日均净流入           | 2.35→1.63            | ❌ 信息重叠                  |

### 8.1 最关键的教训: 中性化管线不兼容

实验 1 是本项目最重要的教训。不是所有数据都适用同一条管线:

- 量价因子 **需要** 中性化（去行业市值噪声 = 提取纯 alpha）
- 权重因子 **不能** 中性化（行业和市值本身就是信号）

**先理解数据的信号结构，再决定处理方法。**

### 8.2 IC ≠ 回测 (三次独立验证)

1. v3 multi-seed: ICIR 升 15% 但夏普降 23%
2. news_count: ICIR 7.29→7.56 但夏普 2.35→1.16
3. PE 滞后: 消除前瞻偏差但夏普 2.35→1.63

**结论**: 全样本相关性不等于 top-K 选股质量。IC 指标对优化有用，但不能作为最终评估标准。

## 10. 核心发现: 7 个反直觉结论

### 发现 1: IC 持平 ≠ 实战收益持平

LGBM 与 GRU 的 Test IC 完全相同 (0.0511)，但回测夏普差 2.6 倍 (2.21 vs 0.80)。
**原因**: GRU 在极端样本上输出极端预测，导致 top-K 排序不稳。IC 度量全样本相关性，不直接度量 top-K 选股质量。

### 发现 2: 横截面建模 >> 单股时序建模

GRU (仅时序) < LGBM (隐式横截面) < MASTER (显式双轴)。
A 股选股的核心是"在 800 只股票中选最好的 10 只"——这是一个横截面排序问题，不是时序预测问题。

### 发现 3: Market Guidance 有效

MASTER 的 market-gate 模块用大盘状态调制原始因子。等参数量下，加入 market gate 的版本明显占优。**个股 alpha 与大盘 beta 解耦后再融合，比直接堆叠特征更有效。**

### 发现 4: A 股短期信号收敛速度快

三个模型都在 epoch 2-6 达到验证集最佳。数据信号清晰强烈，不需要复杂模型，但需要**正确的归纳偏置**。容量竞赛意义不大。

### 发现 5: 加深加宽反而过拟合 (v2 反例)

参数量 122K→422K (3.5x)，夏普 2.24→1.12。**A 股短期 alpha 容量上限约 ~100K 参数。** 数据信号的复杂度有限，更大模型只是拟合噪声。

### 发现 6: IC 长期衰减 + 2026Q2 信号反转

LGBM 全期推理: IC 从 2016 的 0.17 单调降至 2025 的 0.05 (-71%)。MASTER 2026Q2 RankIC = -0.094 信号转负。候选制度断点: 2017 (外资流入) / 2023 (全面注册制) / 2026。**单次切分训练不应用于 > 2 年，必须滚动重训。**

### 发现 7: 真实约束对夏普影响小但对累积净值影响大

年化 -2.4pp，夏普 -5%，但累积净值 -37pp。**年化和夏普才是策略可行性的稳健度量；累积收益易被费率累积放大。**

## 11. 实验教训: 10 条实战心得

### 数据管线 (4 条)

1. **中性化不是万能滤镜** — 先理解数据信号结构，再决定处理方法
2. **FACTOR_COLS 散落 10 个文件** — 引入 config.py 单一数据源解决
3. **Windows GBK 编码是隐形杀手** — tqdm 进度条字符触发 UnicodeDecodeError
4. **数据同步要检查所有目录** — market/ 未同步导致 9 天数据缺失

### 模型架构 (3 条)

5. **更好的特征结构 > 更多的模型** — 单模型+独立通道 (123K) 完胜三模型集成 (369K)
6. **multi-seed 在种子质量不均时是毒药** — 弱 seed 稀释强 seed
7. **IC 提升 ≠ 回测提升** — 三次独立验证

### 特征工程 (3 条)

8. **关键词法情感打分不可用于交易** — 必须用 FinBERT 级别模型
9. **权重因子正确用法** — 月频→日频前向填充 + 不中性化 + 独立通道，三条件缺一不可
10. **先验证简单方案，再叠加复杂度** — v4-v7 全部低于 v3_search

## 12. 未来方向与局限性

### 已知局限

| 维度     | 已做                                                       | 未做                               |
| -------- | ---------------------------------------------------------- | ---------------------------------- |
| 数据治理 | 日横截面中性化 / train段标准化 / ST动态剔除 / 上市天数过滤 | 财报公告日错位（PE 滞后 30-60 天） |
| universe | 月度滚动中证 800 近似                                      | 非严格中证 800 真实成分            |
| 回测     | T+1 + 涨跌停 + 双边费 + ST                                 | 未模拟订单滑点 / 成交量约束        |
| 策略     | 等权 n=10/k=2 简单换手                                     | 未做估值底线/资金流确认/流动性门槛 |
| 训练     | 单次切分                                                   | 未做 walk-forward retraining       |
| 另类数据 | 资金流 + 中性化                                            | 未做 FinBERT 新闻情绪              |

### 可行改进方向 (按优先级)

1. **滚动重训**: 每 6 个月用最新数据重新训练（应对 IC 衰减）
2. **流动性过滤**: 剔除日均成交额最低的 20% 股票（减少滑点）
3. **FinBERT 新闻情绪**: 替代词典法，提升情绪因子有效性
4. **行业中性化持仓**: 限制单行业最大权重（降低集中度风险）
5. **adaptive (n, k)**: 根据市场波动率动态调整持仓数量和换手率

### 实验哲学

> 多次失败实验比单次成功更有信息量。v2 (容量增加) 和 v3 (multi-seed) 都不如 v1，这两个"失败"实际给了关键洞察。诚实而具体地报告衰减（2026Q2 RankIC 转负），不掩盖问题，反而把它作为"模型衰减证据"和"滚动训练必要性"的论据。记录每一步的 **why**，而非 **what**。
